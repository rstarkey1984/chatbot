Chatbot Gateway

AI 대화 문맥 관리를 위한
세션 라우팅 서비스

user+custom 스코프 기반 sessionId 설계와 OpenClaw Gateway 연동

1. 문제 정의

  • 같은 사용자 요청이 하나의 세션으로 섞이면 문맥 충돌이 발생
  • 채널/탭/쓰레드 단위의 대화 분리가 필요
  • 운영 환경에서는 세션 무결성과 라우팅 일관성이 핵심

2. 핵심 아키텍처

  • SessionRoutingService: sessionTicket/default-session 규칙으로 세션 계산
  • ChatQueryService: 검증, 동시성 제어, rate limit
  • GatewayClient: OpenClaw Gateway /v1/chat/completions 호출
  • 응답/로그를 동일 sessionKey로 일관 처리

3. 세션 모델: user + custom

  • sessionId = 사용자 식별자 + custom 식별자
  • custom: 채널/방/탭/쓰레드/업무 컨텍스트 분리 키
  • same user + different custom = 다른 세션 문맥
  • same user + same custom = 기존 세션 문맥 재사용

4. Gateway 요청 계약

  • Authorization: Bearer <gateway-token>
  • x-openclaw-agent-id: chatbot
  • x-openclaw-session-key: {prefix}{sessionId}
  • stream: HttpClient+BufferedReader, non-stream: WebClient

5. 운영 정책 분리

  • 코드(Spring): 검증/라우팅/전달
  • 정책 파일(OpenClaw): 응답 기준/스타일/행동 제한

6. 기대 효과

  • 문맥 혼선 감소: 세션 스코프 명확화
  • 운영 용이성 향상: 세션/정책 책임 분리
  • 확장성 확보: custom 키 추가만으로 컨텍스트 확장
  • 품질 일관성: 동일 sessionKey 기반 응답 누적
